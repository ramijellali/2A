Relations entre les Entités - Système d'Enquêtes de Satisfaction

Vue d'ensemble : Ce système gère des enquêtes de satisfaction avec 5 entités principales liées entre elles par des clés étrangères.

🏗️ Structure des Entités

👥 UTILISATEURS

Rôles : admin, agent, client
Champs clés : id (PK), nom, prenom, email, role, statut
Fonction : Gère tous les acteurs du système

🎯 CAMPAGNES

Relations : created_by → utilisateurs(id)
Champs clés : id (PK), nom, description, date_debut, date_fin, statut
Fonction : Organise les enquêtes par période/thématique

📋 ENQUETES

Relations : campagne_id → campagnes(id), created_by → utilisateurs(id)
Champs clés : id (PK), titre, description, statut
Fonction : Contient les questionnaires spécifiques

❓ QUESTIONS

Relations : enquete_id → enquetes(id)
Champs clés : id (PK), texte, type_question, options_json, ordre_affichage
Types : texte_libre, choix_multiple, notation, oui_non

💬 REPONSES

Relations : question_id → questions(id), utilisateur_id → utilisateurs(id)
Champs clés : id (PK), reponse_texte, reponse_numerique
Contrainte : Une seule réponse par utilisateur/question

📊 Types de Relations

Relations One-to-Many (1:N)

UTILISATEURS → CAMPAGNES
Un utilisateur (admin/agent) peut créer plusieurs campagnes
created_by → utilisateurs(id)
UTILISATEURS → ENQUETES
Un utilisateur (agent) peut créer plusieurs enquêtes
created_by → utilisateurs(id)
CAMPAGNES → ENQUETES
Une campagne peut contenir plusieurs enquêtes
campagne_id → campagnes(id)
ENQUETES → QUESTIONS
Une enquête peut avoir plusieurs questions
enquete_id → enquetes(id)
QUESTIONS → REPONSES
Une question peut avoir plusieurs réponses (de différents clients)
question_id → questions(id)
UTILISATEURS → REPONSES
Un utilisateur (client) peut donner plusieurs réponses
utilisateur_id → utilisateurs(id)

🎯 Contraintes d'Intégrité

Contraintes Importantes

Unicité Réponse :
Un client ne peut répondre qu'une seule fois à une question
UNIQUE KEY unique_user_question (utilisateur_id, question_id)
Suppression en Cascade :
• Supprimer une campagne → supprime ses enquêtes
• Supprimer une enquête → supprime ses questions
• Supprimer une question → supprime ses réponses
ON DELETE CASCADE
Types de Questions :
texte_libre, choix_multiple, notation, oui_non
ENUM('texte_libre', 'choix_multiple', 'notation', 'oui_non')

🔄 Flux de Données

Processus Typique :

  1. Admin crée une CAMPAGNE
  2. Agent crée des ENQUETES dans la campagne
  3. Agent ajoute des QUESTIONS aux enquêtes
  4. Clients donnent des REPONSES aux questions
  5. Admin/Agent analysent les résultats

📈 Requêtes Complexes Courantes

Statistiques par Enquête :
JOIN enquetes → questions → reponses
Réponses d'un Client :
JOIN utilisateurs → reponses → questions → enquetes
Performance des Campagnes :
JOIN campagnes → enquetes → questions → reponses
